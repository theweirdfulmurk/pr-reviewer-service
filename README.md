# PR Reviewer Assignment Service

Микросервис для автоматического назначения ревьюеров на Pull Request'ы.

## Стек

- Go 1.23
- Fiber v2 (HTTP фреймворк)
- gocraft/dbr v2 (инструмент построения SQL запросов)
- lib/pq (драйвер PostgreSQL)
- zap (логгера)
- PostgreSQL 16

## Запуск

```bash
docker-compose up --build
```

Сервис будет доступен на `http://localhost:8080`.

## API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/team/add` | Создать команду с участниками |
| GET | `/team/get?team_name=` | Получить команду |
| POST | `/users/setIsActive` | Установить флаг активности |
| GET | `/users/getReview?user_id=` | PR'ы пользователя как ревьюера |
| POST | `/pullRequest/create` | Создать PR с автоназначением |
| POST | `/pullRequest/merge` | Смержить PR (идемпотентно) |
| POST | `/pullRequest/reassign` | Переназначить ревьюера |
| GET  | `/health` | Health-check сервиса |


## Принятые решения

### 1. Уникальность user_id

`user_id` глобально уникален. При создании команды с существующим пользователем — пользователь обновляется (включая смену команды).

### 2. Переназначение ревьюера

При переназначении новый ревьюер выбирается из команды **заменяемого** ревьюера (не автора PR). Это соответствует спецификации: "заменяет одного ревьювера на случайного активного участника из команды заменяемого ревьювера".

### 3. Выбор ревьюеров

- Исключается автор PR
- Исключаются неактивные пользователи
- При переназначении исключаются текущие ревьюеры PR
- Выбор случайный среди доступных кандидатов

### 4. Идемпотентность merge

Повторный вызов merge для уже смерженного PR возвращает 200 OK с текущим состоянием.

### 5. Пустые списки ревьюеров

Если в команде автора нет активных участников (кроме автора), PR создаётся с пустым списком ревьюеров.

├── internal/
│   ├── config/          # Конфигурация и логгер

│   ├── service/         # Бизнес-логика
